# 사상의 그물망 — 검증 계약을 통과해야만 그려지는 영향 지도

**AI가 조사한 관계를 어디까지 믿을 수 있는가?** 이 저장소는 그 질문에 코드로 답한다. 책 한 권을 입력하면 LLM이 선행 사상과 후대 영향을 공개 웹에서 조사해 양방향 그물망으로 만들되, **모든 연결선은 출처 등급, 독립 발행 주체, 관계 유형별 근거 기준을 코드로 통과해야만 화면에 남는다.** 기준 미달 연결은 예외 없이 자동 제거된다.

문학은 첫 적용 도메인일 뿐이다. 도메인에 속한 부분은 스키마와 프롬프트이고, 구조화 출력→출처 정책 검증→인수 검사로 이어지는 골격은 도메인과 무관하다. 스키마와 출처 정책만 바꾸면 의사결정 이력 추적, 기술 부채의 기원 지도, 문서의 주장-근거 감사에 같은 루프가 그대로 돈다.

사용자는 책 제목과 저자만 입력한다. 생성 단계에서는 ChatGPT로 로그인된 로컬 Codex CLI가 공개 웹 자료를 조사하고, 결과물은 모든 데이터·CSS·JavaScript가 내장된 단일 HTML로 저장된다. 완성된 HTML을 볼 때는 인터넷도 API 키도 필요 없다.

## 이 프로젝트가 푸는 문제

책을 읽다 보면 더 오래된 책과 사상으로 계속 거슬러 올라가게 된다. 반대로 한 작품이 후대의 어떤 작가와 사상에 이어졌는지도 한눈에 보기 어렵다. 이것을 "모든 원전을 완독해야 하는 문제"가 아니라 "전체 계보를 먼저 보여주는 지도가 없는 문제"로 재정의하고, 동시에 그런 지도를 LLM으로 만들 때 반드시 따라오는 두 번째 문제 — **생성된 관계를 신뢰할 근거가 없다는 문제** — 를 검증 계약으로 함께 풀었다.

이를 위해 다음을 구현했다.

- 입력 책을 중심으로 선행·후대 영향 관계를 모두 탐색
- 한 번의 별 모양 검색이 아니라, 발견된 각 노드를 전용 호출로 조사하는 다단계 확장
- 직접 인용·사상적 영향·비판적 계승을 서로 다른 선과 색으로 표현
- 모든 연결에 설명, 근거 요약, 원문 자료 URL과 출처 등급 저장
- 노드 클릭 시 그 노드의 조상과 후손 경로만 강조
- 검색, 확대·축소, 드래그 이동, 전체 화면 맞춤을 지원하는 SVG 네트워크
- 허용되지 않은 출처와 근거가 부족한 해석적 연결을 자동 제거

## 요구 사항

- macOS/Linux, Python 3.10 이상
- OpenAI Codex CLI
- `codex login`을 통해 완료한 ChatGPT 로그인

ChatGPT Plus 로그인 경로를 사용하므로 별도의 OpenAI API 키나 유료 API 호출 코드는 없다. Plus 구독과 OpenAI API 과금은 별개다.

```bash
codex login status
python3 --version
codex --version
```

## 실행

저장소 루트에서 실행한다.

```bash
python3 -m src.cli \
  --title "죄와 벌" \
  --author "표도르 도스토옙스키" \
  --output artifacts/crime-and-punishment.html \
  --rounds 3 \
  --timeout 2400
```

라이브 웹 조사가 기본값이다. 각 라운드는 아직 전용 확장 조사를 받지 않은 노드를 FIFO 큐에서 꺼내 조사하며, 시간 또는 누적 지도 안전 상한에 도달하면 중지한다. 개별 Codex 조사 응답은 50노드·80연결로 제한하고, 클릭으로 누적되는 개인 지도는 최대 200노드·320연결까지 보존한다. `--rounds`를 늘리면 더 깊게 탐색하지만 실행 시간도 길어진다.

재귀 라운드는 단순히 전체 그래프를 다시 요약하지 않는다. 각 노드는 연결 개수와 무관하게 최소 한 번 독립적인 웹 조사 대상으로 처리된다. 따라서 `니체 → 푸코` 하나가 먼저 발견돼도 니체 조사가 완료된 것으로 간주하지 않으며, 니체 전용 호출에서 추가 후대 관계를 찾는다. 오래 기다린 노드는 새 가지보다 먼저 처리되도록 FIFO 순서를 보존한다.

완료 후 생성되는 파일:

- `artifacts/crime-and-punishment.html`: 오프라인 단일 HTML 그물망
- `artifacts/crime-and-punishment.json`: 조사 체크포인트와 전체 근거 데이터
- `artifacts/crime-and-punishment-status.json`: 실행 상태·호출 기록

HTML은 Finder에서 더블클릭해 열 수 있다.

### 노드를 클릭해 다시 분석·확장하기

단일 HTML 더블클릭 모드는 재현 가능한 오프라인 열람용이라 브라우저가 Codex CLI를 실행하거나 JSON을 수정할 수 없다. 클릭 확장이 필요할 때는 저장소 루트에서 로컬 확장 모드를 실행한다.

```bash
python3 -m src.server \
  --checkpoint artifacts/crime-and-punishment.json \
  --html artifacts/crime-and-punishment.html \
  --expansion-rounds 3 \
  --expansion-width 2
```

브라우저가 `http://127.0.0.1:8765/`을 열면 노드를 선택한 뒤 상단의 `다시 분석·확장` 버튼을 누를 수 있다. 로컬 서버는 선택 노드를 새로운 지역 기준점으로 삼아 그 노드의 선행과 후대를 각각 독립 조사한다. 근거 기준을 통과한 결과는 원래 입력 책을 영구 앵커로 유지한 기존 JSON에 누적 병합한다. 화면만 선택 노드를 `확장 기준`으로 다시 배치하고 그 위치로 자동 이동하므로, 새 가지를 바로 확인하면서 최초 입력 책까지의 경로도 보존할 수 있다.

클릭 확장은 기본적으로 3개 재귀 라운드를 사용한다. 첫 라운드는 선택 노드의 선행·후대를 조사하고, 다음 라운드들은 그때 실제로 새로 발견된 노드만 FIFO 큐에 넣어 방향별로 번갈아 확장한다. 따라서 한 단계 이웃에서 멈추지 않으면서도 기존 지도의 무관한 노드를 다시 조사하지 않는다. `--expansion-rounds`와 `--expansion-width`를 높이면 더 넓고 깊게 조사하지만 Codex 호출 수와 실행 시간도 증가하며, 전체 작업은 `--timeout` 안에서 중지된다.

새 노드나 연결이 생긴 경우에만 화면을 새로고침하며, 선택 상태·강조·화면 중심을 복원하고 추가된 노드·연결 수를 표시한다. 검증 기준을 통과한 새 관계가 없으면 새로고침하지 않고 `새 연결이 없습니다`라고 명시한다. 서버는 외부 인터페이스가 아닌 `127.0.0.1`에만 바인딩하며 종료는 실행한 터미널에서 `Control+C`다.

### GitHub Pages용 정적 데모

현재 체크포인트를 정적 데모로 복사하려면 다음 명령을 실행한다. 스크린샷이나 지원서에 특정 확장 상태를 첨부한다면, **그 상태의 체크포인트로 이 명령을 실행해 데모와 노드·연결 수를 일치시킨다.**

```bash
python3 thought_lineage_svg_renderer.py \
  artifacts/crime-and-punishment.json \
  docs/index.html
```

`docs/index.html`은 검색·선택·강조·확대·이동이 가능한 단일 HTML이다. GitHub Pages 같은 정적 호스팅에서는 Codex 프로세스를 실행할 수 없으므로 `다시 분석·확장`은 로컬 서버 모드에서만 제공하며, 이 명령은 그 사실을 알리는 안내 문구를 데모 페이지 상단에 자동으로 넣는다(`--no-demo-note`로 생략 가능). `src.cli`와 로컬 서버가 만드는 산출물에는 이 문구가 들어가지 않는다.

**이렇게 읽어보세요.** 데모에서 `뫼르소, 살인 사건`(2013)을 클릭하면 강조되는 조상 경로가 이 지도의 요지를 요약한다: 루소의 『고백록』(1782)에서 『지하로부터의 수기』(1864)와 『죄와 벌』(1866)을 지나 카뮈의 『이방인』(1942)으로, 다시 알제리 작가 카멜 다우드의 되받아쓰기(2013)로 이어지는 231년의 응답 사슬이다. 각 연결선을 클릭하면 그 관계를 화면에 남게 한 근거 요약과 출처 등급이 나온다.

배포 확인 절차: ① 저장소 push 후 GitHub Settings → Pages에서 `main` 브랜치의 `/docs` 폴더를 소스로 지정 ② 발급된 URL을 시크릿 창에서 열어 그래프 로딩 확인 ③ 모바일 폭(개발자 도구 375px)에서 한 번 열어 레이아웃 확인.

### 누락 후보만 추가 조사하기

영향 관계 검색은 완전한 색인이 아니므로 중요한 인물이 한 번의 생성에서 빠질 수 있다. 기존 결과를 버리지 않고 후보만 검증해 합칠 수 있다. `--focus`는 앵커와의 직접 연결을 강요하지 않는다. 기존 그물망의 모든 노드를 출발 후보로 검사해 가장 짧은 근거 경로와 필요한 중간 노드를 추가한다. 예컨대 현재 그래프에 니체가 있다면 `죄와 벌 → 니체 → 쿤데라`처럼 연결하며, 근거 없는 `죄와 벌 → 쿤데라` 선을 만들지 않는다.

```bash
python3 -m src.cli \
  --title "죄와 벌" \
  --author "표도르 도스토옙스키" \
  --output artifacts/crime-and-punishment.html \
  --resume artifacts/crime-and-punishment.json \
  --focus "Milan Kundera" \
  --rounds 1 \
  --timeout 1200
```

예를 들어 쿤데라는 단순한 ‘영향받은 후대 작가’로 미리 확정하지 않는다. 자료가 도스토옙스키에 대한 의식적 거부나 대립을 뒷받침하면 `critical_response`로 분류하고, 서로 독립적인 A/B 출처 두 곳을 확보하지 못하면 추가하지 않는다.

## 테스트와 검증

```bash
python3 -m unittest discover -s tests -v
python3 scripts/acceptance.py
```

단위 테스트와 기본 인수 검사는 네트워크·모델 결과에 좌우되지 않도록 `--fixture`의 고정 검증 자료를 명시적으로 사용한다. 실제 조사까지 포함한 검사는 다음처럼 별도로 실행한다.

```bash
python3 scripts/acceptance.py --live --rounds 3
```

검증기는 다음을 확인한다.

- 개별 조사 응답과 누적 지도의 크기 상한, 고아 노드 금지
- 직접 관계는 A/B 출처 1개 이상
- 해석적 관계는 서로 다른 발행 주체의 A/B 출처 2개 이상
- 블로그, 위키백과, 검색 결과 페이지, 미분류 도메인 제거
- 출처 도메인 정책보다 과도하게 선언된 등급 자동 하향
- HTML 안에 외부 스크립트·스타일·폰트·런타임 `fetch`가 없는지 검사
- 토큰, 쿠키, API 키 형태의 문자열이 저장소에 남지 않았는지 검사

출처의 존재 여부와 문장이 실제 주장을 뒷받침하는지는 별개의 문제다. 따라서 자동 검증은 출처 정책과 구조적 완전성을 담당하고, 최종 제출 전 중요한 연결은 사람이 원문을 열어 확인한다. LLM이 생성한 계보에는 사실 오류나 누락 가능성이 있다는 안내를 UI에 유지한다.

## 주요 옵션

| 옵션 | 기본값 | 설명 |
|---|---:|---|
| `--title` | 필수 | 중심 책 제목 |
| `--author` | 필수 | 중심 책 저자 |
| `--output` | 필수 | 생성할 HTML 경로 |
| `--rounds` | `3` | 확장 큐를 처리하는 최대 라운드 수 |
| `--frontier-width` | `4` | 라운드마다 별도 Codex 호출로 조사할 노드 수 |
| `--timeout` | `2400` | 전체 실행 제한(초) |
| `--seed` | `1866` | 결과 재현성을 돕는 시드 |
| `--fixture` | 꺼짐 | 라이브 조사 대신 테스트용 고정 데이터 사용 |
| `--resume` | 없음 | 기존 체크포인트에 조사 결과 병합 |
| `--focus` | 없음 | 반드시 검토할 누락 후보(반복 지정 가능) |

상대 출력 경로는 현재 작업공간 안에서만 허용한다. 시간 초과나 조사 실패가 발생해도 가능한 데이터는 `partial` 또는 `timed_out` 체크포인트와 HTML로 보존한다.

## 설계상 선택

- Crossref는 현대 학술 논문의 DOI 인용망이고 Google Books는 본문·서지 검색 중심이어서 고전 사상사의 영향 계보를 그대로 제공하지 않는다. 이 프로젝트는 생성 시점에 Codex의 웹 조사와 LLM 판단을 사용한다.
- 주제가 비슷하다는 이유만으로 연결하지 않는다. 직접 인용·언급과 학술적으로 논의되는 영향·비판적 계승만 후보로 삼고 관계별 근거를 저장한다.
- 입력 책은 계보의 마지막이 아니다. 화면 중앙의 기준점이며, 과거 방향과 미래 방향을 모두 확장한다.
- 완성 HTML에는 조사 결과를 내장한다. 열람 시 사용자 자격 증명, 인터넷, 서버가 필요 없다.

## 프로젝트 구조

```text
src/                         조사·정제·검증·렌더링 파이프라인
schema/                      Codex 구조화 출력 JSON Schema
tests/                       파이프라인·안전·오프라인 UI 테스트
scripts/acceptance.py        재현 가능한 인수 검사
thought_lineage_svg_renderer.py  SVG 그물망 UI 템플릿
artifacts/                   생성 결과와 체크포인트
docs/                        출처 정책과 반복 개선 기록
```

이 저장소의 핵심은 특정 책의 정답 데이터를 하드코딩한 데모가 아니라, **모호한 독서의 답답함을 검증 가능한 조사·정제·시각화 계약으로 바꾼 에이전트 작업 흐름**이다.

구체적인 실패→진단→수정→회귀 테스트 연결은 [`docs/iteration-log.md`](docs/iteration-log.md)에 기록했다.
