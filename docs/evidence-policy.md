# 증거 및 순위 정책

## 출처 허용

허용은 `src/source_validator.py`의 `ALLOWED_DOMAINS`에 도메인과 발행 주체 유형이 명시된 경우로 제한한다. 목록에 없는 도메인은 `unclassified_domain`으로 거부한다. 검색결과 페이지, 리디렉션 래퍼, 개인 블로그, 위키·포럼·소셜 미디어, 출처 없는 요약, AI 생성물, 제휴성 목록은 허용하지 않는다. 하위 도메인은 정확한 도메인 경계로만 인정하여 접미사 위장을 막는다.

등급은 결정론적으로 A(원문/동료심사/대학 출판), B(편집 백과사전·공공 문화기관·주요 언론), C(기타 허용 전문 출처)다. 직접 인용·명시적 언급(`explicit`, `citation`, `adaptation`)은 A/B 출처 1개 이상, 해석적 관계(`thematic`, `ideological`, `critical_response`, `movement`)는 서로 다른 등록 도메인/발행 주체의 A/B 출처 2개 이상이어야 한다.

URL 감사는 모든 고유 URL에 실제 HEAD 요청을 보내고 실패하면 GET으로 재시도하며 상태 코드, 최종 URL, 방식, UTC 검사 시각과 오류를 기록한다. URL 접근 성공은 해당 원문이 저장된 주장을 뒷받침한다는 뜻이 아니다.

표본 원문 대조는 자동 판정하지 않으며 기본 인수 검사에도 사람 검토 완료를 주장하지 않는다. 사람이 직접 검토한 별도 JSON을 `scripts/acceptance.py --human-reviews <path>`로 제공할 때만 선택적으로 집계한다. 이 파일의 각 항목에는 `review_type: "human"`, 검토자, 검토 시각, URL, 페이지·절·문단 위치, 짧은 원문 발췌와 `supports|does_not_support` 판정이 모두 있어야 한다. AI가 수행한 검토는 사람 검토로 집계하지 않는다.

## 중요도와 동점

중요도(0–100)는 명시성, 앵커 관련성, 비평 합의, 역사적 파급을 함께 고려해 프롬프트가 정수로 부여하고 검증 시 범위를 확인한다. 정렬은 `(신뢰도 A>B>C, 중요도 내림차순, source id, target id, relation id)` 순이다. 중복은 같은 시작·끝·관계 유형을 합치고 URL 기준으로 출처를 합친다. 한도 초과 시 이 순서대로 최대 80개 연결과 그 연결에 필요한 최대 50개 노드를 보존하고, 연결 없는 비앵커 노드를 제거한다.
